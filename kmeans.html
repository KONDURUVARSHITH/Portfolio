Iris Dataset Clustering Analysis

Code Repository

Project Overview

This project implements and compares different clustering algorithms (K-means and Hierarchical) on the Iris dataset. The analysis includes data preprocessing, visualization, clustering implementation, and performance evaluation using silhouette scores.

Dataset Description

  • 150 samples of iris flowers
  • 4 features: sepal length, sepal width, petal length, petal width
  • 3 different species: Setosa, Versicolor, Virginica

Implementation Steps

1. Data Preprocessing

  • Loaded data using pandas
  • Applied StandardScaler for feature normalization
  • Created visualization using seaborn's pairplot

Key Findings

  • Both algorithms performed best with 2 clusters (Silhouette scores ~0.58)
  • K-means slightly outperformed Hierarchical Clustering for most cluster counts
  • Performance degraded as number of clusters increased beyond 5
  • 3 clusters (matching species count) showed moderate performance (scores ~0.45)

Silhouette Score Analysis

K-means Score Range:

  • Best: 0.582 (K=2)
  • Moderate: 0.460 (K=3)
  • Lowest: 0.289 (K=26)

Hierarchical Score Range:

  • Best: 0.577 (n_clusters=2)
  • Moderate: 0.447 (n_clusters=3)
  • Lowest: 0.284 (n_clusters=26)

Conclusions

The analysis revealed that both K-means and Hierarchical Clustering algorithms performed similarly, with K-means showing slightly better results overall. The optimal number of clusters appears to be 2 or 3, with diminishing returns as cluster count increases. The natural grouping in the data seems to favor two main clusters, though three clusters still provide meaningful separation corresponding to the three iris species.
